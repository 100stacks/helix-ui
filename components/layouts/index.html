Layouts

WARNING Layouts are not automatic. Apply appropriate CSS class to document body.

WARNING Padding is not automatic. Add Box styling to apply padding.

Two layouts are officially supported, and both have the same basic HTML structure.

  • The Head is reserved for Pilot navigation.
  • The Foot is reserved for legal information.
  • The Stage is the area between the Head and Foot where you'll do most of your work.

Vertical Layout

Use a Vertical layout if your application content needs to grow along the y-axis (pushing the footer down past the bottom of the viewport).

  • Apply the .hxVertical class to the document body (<body class="hxVertical">).
  • Vertical Template

Horizontal Layout

Use a Horizontal layout if your application content needs to grow across the x-axis (keeping the footer visible at all times).

  • Apply the .hxHorizontal class to the document body (<body class="hxHorizontal">).
  • Horizontal Template

Stage

WARNING Do not add more than one side rail in your layout.

The stage is the area between the head and footer. It houses the applicaton content, left-hand navigation, and side rail containers.

  • Give the stage an id of stage to apply layout CSS.

Left-Hand Navigation

  • For accessibility reasons, use a <nav> element.
  • Give the element an id of nav to apply layout CSS. 
<nav id="nav">...</nav>

Application Content

INFO Content area behaves like a non-wrapping Grid row when using a Horizontal layout.

  • For accessibility reasons, use a <main> element.
  • Give the element an id of content to apply layout CSS. 
<main id="content">...</main>

Side Rail

OPTIONAL Main content will automatically fill the space of a missing side rail.

WARNING A side rail is not recommended for a Horizontal layout.

  • For accessibility reasons, use an <aside> element.
  • Give the element a class of hxSiderail to apply layout CSS. 
<aside class="hxSiderail">...</aside>
